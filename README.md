1. Подготовка среды

Скачайте все файлы на рабочий стол.

Установите интерпретатор Python (версии 3.10 или выше).

Используйте любую удобную среду разработки: PyCharm, VS Code или стандартный терминал.

Откройте папку с файлами проекта и запустите main.py в вашем редакторе.

2. Установка необходимых компонентов
Откройте встроенный терминал и выполните установку библиотек одной командой:

pip install streamlit groq python-docx pypdf

Дождитесь завершения загрузки всех пакетов. 
Это займет буквально 5-10 мин в зависимости от скорости интернета.

3. Запуск приложения
В терминале введите команду для старта веб-интерфейса:

streamlit run main.py

При первом запуске разрешите доступ к сети в окне брандмауэра Windows, если появится соответствующий запрос.

Порядок работы с программой здесь есть но в README2 расписано более подробней.
Авторизация: В поле «Имя и номер ТЗ» введите идентификатор проекта, а в поле «API Ключ» — ваш персональный ключ доступа Groq (начинается с gsk_).
Ключ-его можно будет менять по желанию или при нехватки токенов.

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    (на нем есть гдето 20 загрузок но зато бесплатных) если потратите можно поменять акаунт и еще тестить делается за 5 минут.

Загрузка регламента: Выберите файл промпта (prompt.txt). Вы можете использовать готовый вариант из нашего репозитория на GitHub или загрузить свой собственный алгоритм проверки. Его можно будет изменять и полностью менять для доработки регламента. Не залезая в сам код.

Загрузка документа: Выберите файл Технического задания в формате .docx, .pdf или .txt, который необходимо подвергнуть аудиту.

Анализ: Нажмите на красную кнопку «ВЫПОЛНИТЬ ПРОВЕРКУ».

Результат: Система мгновенно выведет визуальный статус соответствия (✅/❌) и сформирует детальное текстовое заключение.


# spec-ai-check
Текстовое описание

Постановка проблемы:
В процессе подготовки технических заданий часто возникают ошибки, связанные с отсутствием обязательных разделов, логическими несоответствиями между задачами и результатами, а также некорректным оформлением финансовых показателей. Это приводит к снижению качества документации и увеличению времени на её проверку.

Предлагаемое решение:
Разрабатывается веб-приложение, предназначенное для автоматической проверки технических заданий на соответствие установленным требованиям. Система выполняет анализ структуры документа, выявляет логические ошибки, проверяет наличие обязательных разделов и корректность формулировок.

Описание применения искусственного интеллекта:
Искусственный интеллект используется для анализа текстового содержания технических заданий. Модель выявляет несоответствия между задачами и результатами, обнаруживает отсутствующие разделы, а также определяет нарушения в формулировках и структуре документа. AI выполняет семантический анализ текста, а не только формальную проверку.

Характеристика целевой аудитории системы:
Целевой аудиторией являются студенты, преподаватели, научные сотрудники, а также специалисты, занимающиеся подготовкой технической документации и проектов. Система ориентирована на пользователей, которым необходимо быстро и качественно проверять ТЗ на соответствие требованиям.

Оценка возможности внедрения:
Система может быть внедрена как веб-сервис с использованием современных технологий (Python, Streamlit, API искусственного интеллекта). Для её функционирования требуется доступ к интернету и API модели. Реализация не требует сложной инфраструктуры и может быть развернута на облачных платформах.

Перспективы коммерциализации:
Проект имеет потенциал коммерциализации в виде SaaS-платформы для автоматической проверки технических заданий. Возможна монетизация через подписочную модель, корпоративные лицензии для образовательных учреждений и интеграцию в системы управления документами.
